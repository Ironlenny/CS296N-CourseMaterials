Bower

Bower is a package manager

  • Package managers are widely used in software development. Some examples are:
    • NuGet for Visual Studio
    • NPM for Node.js and for JavaScript libraries in general
    • Linux distributions each have their own package managers: rpm for Red Hat, yum for Debian
  • Why an additional package manager (besides NuGet and NPM)?
    • Bower is widely used outside the .NET world - it is a standard tool. It is designed specifically for managing client-side packages.
    • NuGet doesn't need to duplicate the functionality of Bower and it's community is already maintaining the packages, so this effort does not need to be duplicated.
    • NPM is primarily designed for Node.js modules, but can be used for front end packages with the addition of Browserify.
  • Official web site: https://bower.io

What Bower does

It's greatest benefit is that it simplifies management of versions and dependencies of our front-end packages
  • When a package is added to the package list, Bower downloads the package and installs it in our project
    • If a package in the list has dependencies (uses other packages), Bower downloads and installs those as well.
    • In ASP.NET Core projects, that means the packages are downloaded and un-zipped into wwwroot/lib
  • When packages are removed from the list, Bower deletes the files from our project.
    • If the package had dependencies, and those are not being used by another package in our project, they will be deleted as well.
  • When package version numbers are changed, Bower replaces the files in our project with the specified version.
    • The appropriate dependent packages are replaced as well

The Bower package list

  • In a JSON file, bower.json
  • The first three entries are the default of an ASP.NET Core project. The dependencies object describes the packages we want to provide for the client (browser)
  • Example:
    {
       "name": "asp.net",
       "private": true,
       "dependencies": {
          "bootstrap": "3.3.7"
       }
    }
  • Bower package repository: http://bower.io/search

The future of Bower

  • Deprecated in Visual Studio-- NuGet already can manage most front-end packages and will soon be the default front-end package manager in Visual Studio
  • Deprecated outside of Visual Studio as well-- by it's own creators! - https://snyk.io/blog/bower-is-dead/
  • Replacements:
    • NuGet
    • Yarn
    • Webpack

Bundling and Minification

  • Facilitate fast file loading for client browsers
    • Bundling - reduce the number of HTTP requests needed
    • Minification - reduce file size
  • Visual Studio Bundler & Minifier extension
    • Add to visual studio through the menu item: Tools ➤ Extensions and Updates, then select "online"
    • In the Solution Explorer, select multiple files of the same type (css or js), right-click, and select "Bundle and MInify Files"
      • Files must be selected in the order you want them to be loaded by the browser
      • The result will be a bundle.css or bundle.js file which will contain:
        • bundle.min.js
        • or bundle.min.css
    • Experiment: Open the new files and inspect them. What do you see?
    • Visual Studio automatically updates minified bundles
      • Modify a source .js or .css file to see how the bundle.min file gets updated
  • Linking to the files in the Razor View
    • Modify the original link and script elements to link to the new minified bundles:
      • <link rel="stylesheet" href="css/bundle.min.css" />
      • <script src="js/bundle.min.js"></script>
